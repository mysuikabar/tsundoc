# tsundoc 要件定義書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | tsundoc |
| 目的 | 読みたい本のストック管理、読書状況の追跡 |
| 概要 | ISBN を入力するだけで書籍情報を自動取得し、「未読 → 読書中 → 読了」のステータスで積読を管理できる Web アプリ |

## 2. 技術構成

| レイヤー | 技術 | 選定理由 |
|----------|------|----------|
| フロントエンド / バックエンド | Next.js (App Router) + TypeScript | フルスタックを 1 リポジトリで完結 |
| データベース | Cloudflare D1 (SQLite ベース) | Edge で動作、無料枠が十分 |
| ORM | Drizzle ORM | D1 との親和性が高く、型安全 |
| 認証 | Better Auth | メール+パスワード認証をシンプルに実装可能 |
| デプロイ | Cloudflare Pages | Edge 配信で高速、D1 との統合が容易 |
| 書籍情報 API | OpenBD（主）/ Google Books API（副） | OpenBD は国内書籍に強く無料。見つからない場合のフォールバックとして Google Books API を使用 |

## 3. MVP 機能要件

### 3.1 認証

- ユーザー登録（メールアドレス + パスワード）
- ログイン
- ログアウト

### 3.2 書籍登録

- ISBN を入力して書籍情報を API 経由で自動取得
- 取得した情報（タイトル・著者・出版社・表紙画像など）を確認して登録
- API で情報が見つからない場合はエラーメッセージを表示

### 3.3 積読リスト

- 登録した書籍の一覧表示
- ステータスによるフィルタリング（全て / 未読 / 読書中 / 読了）

### 3.4 ステータス管理

- 3 段階のステータス: **未読** / **読書中** / **読了**
- リスト上からステータスを変更可能

### 3.5 書籍削除

- リストから書籍を削除

## 4. 将来の拡張機能（MVP 対象外）

- 読書メモ・感想の記録
- カテゴリ・タグによる分類
- 読書統計（読了数の推移、ジャンル別集計）
- URL 共有からの書籍登録
- バーコードスキャンでの登録

## 5. データモデル

### 5.1 users テーブル

Better Auth が管理するテーブル。スキーマは Better Auth の規約に従う。

### 5.2 books テーブル

| カラム | 型 | 制約 | 説明 |
|--------|-----|------|------|
| id | TEXT | PK | ULID or UUID |
| isbn | TEXT | UNIQUE, NOT NULL | ISBN-13 |
| title | TEXT | NOT NULL | 書籍タイトル |
| author | TEXT | | 著者名 |
| publisher | TEXT | | 出版社 |
| cover_image_url | TEXT | | 表紙画像の URL |
| description | TEXT | | 書籍の説明 |
| created_at | TEXT | NOT NULL | 作成日時 (ISO 8601) |

### 5.3 user_books テーブル

| カラム | 型 | 制約 | 説明 |
|--------|-----|------|------|
| id | TEXT | PK | ULID or UUID |
| user_id | TEXT | FK → users.id, NOT NULL | ユーザー ID |
| book_id | TEXT | FK → books.id, NOT NULL | 書籍 ID |
| status | TEXT | NOT NULL, DEFAULT 'unread' | unread / reading / read |
| created_at | TEXT | NOT NULL | 登録日時 (ISO 8601) |
| updated_at | TEXT | NOT NULL | 更新日時 (ISO 8601) |

- `(user_id, book_id)` に UNIQUE 制約を設定

## 6. 画面一覧

### 6.1 ログイン / 新規登録画面

- メールアドレスとパスワードの入力フォーム
- ログインと新規登録の切り替え

### 6.2 積読リスト画面（メイン画面）

- 登録済み書籍の一覧（表紙画像・タイトル・著者・ステータス）
- ステータスによるフィルタタブ（全て / 未読 / 読書中 / 読了）
- ステータス変更操作
- 書籍削除操作
- 書籍登録画面への導線

### 6.3 書籍登録画面

- ISBN 入力フォーム
- 検索結果のプレビュー（タイトル・著者・表紙画像）
- 登録確認ボタン

## 7. 非機能要件

- **レスポンシブ対応**: モバイルファーストで設計し、スマートフォンでも快適に操作できる UI
- **パフォーマンス**: Cloudflare Edge での配信により高速なレスポンスを実現
- **データ整合性**: 同一 ISBN の書籍は books テーブルで重複させず、user_books で紐付ける
